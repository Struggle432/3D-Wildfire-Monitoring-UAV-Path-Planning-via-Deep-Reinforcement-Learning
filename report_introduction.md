# 基于深度强化学习的3D野火场景下无人机监测路径规划

## 实验报告

---

## 1. 网络模型与基本假设

### 1.1 运动环境

本实验构建了一个**三维网格地图环境**，用于模拟无人机在野火场景中的监测任务。

- **空间维度**：三维空间，地面为30×30的二维网格，高度方向分为8层（z=1~8）
- **火势模型**：采用元胞自动机（Cellular Automata）在二维地面网格上模拟火势蔓延
  - 每个格子具有火势强度属性（0~1），0表示无火，1表示火势最盛
  - 火势每步以一定概率向8邻域扩散，扩散概率受风向和风速影响
  - 顺风方向扩散概率高（0.27），逆风方向低（0.03），侧面居中（0.12）
  - 火势强度随时间衰减（衰减率0.015/步），模拟燃烧消耗
- **风场**：持续风，具有方向和速度两个参数，影响火势蔓延的方向和速度
- **障碍物**：强火区域（强度>0.7）视为危险区域，无人机应避免低空穿越

### 1.2 无人机运动模型

- **类型**：旋翼无人机（多旋翼），可悬停、垂直起降
- **动作空间**：7个离散动作——北、南、西、东、上升、下降、悬停观测
- **运动约束**：
  - 水平移动：每步移动1格，成本0.1
  - 垂直移动：每步升降1层，成本0.2（升降能耗大于平移）
  - 转弯：改变水平方向时额外成本0.1（转弯>直行）
  - 悬停：成本0.02（最低能耗状态）
- **起始位置**：地图左下角(2,2,4)，高度4层

### 1.3 观测模型

无人机的观测能力随高度变化，体现**高度-信息量-安全性**的三方权衡：

| 高度 | 观测半径 | 观测精度 | 安全性 |
|------|---------|---------|--------|
| 低(z=1~2) | 4~5格 | 高（逐格观测） | 低（距火近） |
| 中(z=3~5) | 6~8格 | 中（部分遗漏） | 中 |
| 高(z=6~8) | 9~11格 | 低（稀疏采样） | 高（距火远） |

- 观测半径 = 基础半径(3) + 高度×1
- 高空(z>3)观测时，每个格子有15%概率遗漏
- 局部观测图尺寸：9×9，高空时采样步长增大

### 1.4 障碍物分布与火源设定

- 默认2个火源，位于地图中心区域（1/4~3/4范围）
- 每个火源初始占3×3区域，中心强度1.0，边缘递减
- 火势随时间动态蔓延，形成不断扩大的燃烧区域

## 2. 算法的具体设计

### 2.1 算法选择：PPO（Proximal Policy Optimization）

PPO是当前最流行的深度强化学习算法之一，具有训练稳定、超参数鲁棒、样本效率较高等优点，适合连续决策的路径规划任务。

**核心思想**：在策略梯度的基础上，通过裁剪目标函数限制策略更新幅度，避免更新过大导致性能崩溃。

### 2.2 网络结构

- **策略网络π**：MLP，2层256神经元，输出7个动作的概率分布
- **价值网络V**：MLP，2层256神经元，输出状态价值估计
- **激活函数**：ReLU
- **优化器**：Adam，学习率3×10⁻⁴

### 2.3 观测空间设计

观测向量为272维，包含：

| 分量 | 维度 | 说明 |
|------|------|------|
| UAV位置 | 3 | (x/W, y/H, z/Z_max) 归一化 |
| 局部火势图 | 81 | 9×9局部火势强度 |
| 局部覆盖图 | 81 | 9×9已观测标记 |
| 全局火势概览 | 100 | 10×10下采样全局火势 |
| 风场信息 | 2 | 风速×风向分量 |
| 标量特征 | 5 | 火前沿距离、覆盖率、未观测比例、上一步动作、时间步 |

### 2.4 奖励函数设计

```
R = α·信息增益 + β·前沿覆盖 - γ·运动成本 - δ·安全风险 - ε·遗漏惩罚 + ζ·高度奖励 - η·时间惩罚
```

| 分量 | 权重 | 物理含义 |
|------|------|---------|
| 信息增益 | α=2.0 | 本步新观测到的燃烧格子数，鼓励探索未知火区 |
| 前沿覆盖 | β=1.0 | 观测到火势前沿（燃烧边界）的比例，优先监测火线 |
| 运动成本 | γ=0.15 | 转弯>直行，升降>平移，鼓励高效路径 |
| 安全风险 | δ=3.0 | 距火源过近的指数惩罚，低空加倍，保障飞行安全 |
| 遗漏惩罚 | ε=0.05 | 未观测燃烧区占比，减少监测盲区 |
| 高度奖励 | ζ=0.3 | 高斯分布，最优高度4层，权衡观测与安全 |
| 时间惩罚 | η=0.02 | 每步固定惩罚，鼓励尽快完成任务 |

### 2.5 训练流程伪代码

```
输入：环境Env，PPO超参数
输出：训练好的策略π_θ

1. 初始化策略网络π_θ和价值网络V_φ
2. 创建4个并行环境Env_1, ..., Env_4
3. for iteration = 1 to N do
4.     在各并行环境中收集n_steps=2048步转移数据
5.     计算GAE优势估计：A_t = Σ (γλ)^l δ_{t+l}
6.     for epoch = 1 to 10 do
7.         随机采样batch_size=64的小批量
8.         计算裁剪策略梯度损失L_CLIP
9.         计算价值函数损失L_VF
10.        计算熵正则化L_ENT
11.        更新θ, φ：minimize L_CLIP + c_1·L_VF - c_2·L_ENT
12.    end for
13.    每10000步评估一次，保存最佳模型
14. end for
15. return π_θ
```

### 2.6 基线算法

为验证PPO的优势，实现三种基线对比：

1. **随机游走**：每步随机选择动作
2. **贪心策略**：每步评估所有7个动作的期望信息增益，选择最优
3. **A*规划**：A*搜索到最近未观测火区的路径，逐步重规划

## 3. 算法仿真与性能评估

### 3.1 仿真环境设置

| 参数 | 值 | 说明 |
|------|-----|------|
| 地图尺寸 | 30×30 | 地面网格 |
| 高度层数 | 8 | z=1~8 |
| 火源数量 | 2 | 默认双火源 |
| 风速 | 0.6 | 归一化0~1 |
| 风向 | (0.89, 0.45) | 东北偏东 |
| 最大步数 | 300 | 每回合上限 |
| 训练总步数 | 500,000 | PPO训练 |
| 并行环境 | 4 | SubprocVecEnv |
| GPU | CUDA | 自动检测 |

### 3.2 训练过程

- 训练设备：NVIDIA GPU (CUDA)
- 训练时长：约70分钟
- 收敛情况：约200K步后奖励趋于稳定（~1300）
- 最终评估奖励：1261.39 ± 36.69

### 3.3 算法对比结果

| 算法 | 平均奖励 | 奖励标准差 | 火区覆盖率 | 平均步数 |
|------|---------|-----------|-----------|---------|
| **PPO** | **1317.84** | 124.02 | **85.95%** | 190 |
| 贪心策略 | 1299.46 | 84.94 | 90.56% | 88 |
| A*规划 | 1186.46 | 83.62 | 90.33% | 147 |
| 随机游走 | 192.80 | 174.23 | 22.39% | 115 |

**分析**：
- PPO在总奖励上最优，说明其在信息增益与运动成本的权衡上表现最佳
- 贪心策略和A*覆盖率略高，但步数更少（更早结束），总奖励较低
- 随机游走几乎无法有效监测火区，覆盖率仅22%

### 3.4 多场景评估

| 场景 | 奖励 | 覆盖率 |
|------|------|--------|
| 单火源无风 | 1297.98 | 79.92% |
| 单火源有风 | 1256.97 | 83.78% |
| 双火源有风 | 1271.43 | 83.14% |
| 三火源强风 | 1173.09 | 78.87% |

**分析**：
- 有风场景覆盖率高于无风，因为风向使火势定向蔓延，更易预测和追踪
- 多火源场景奖励略降，但策略仍能有效分配监测资源
- 强风场景最具挑战性，火势蔓延快且范围大

## 4. 结论与未来展望

### 4.1 结论

1. **PPO算法能够有效学习3D野火监测策略**：在动态火场环境中，PPO训练的无人机能够自主规划路径，实现85%以上的火区覆盖率，显著优于随机基线。

2. **奖励函数设计是关键**：多目标奖励函数（信息增益、前沿覆盖、运动成本、安全风险、遗漏惩罚、高度权衡）的合理设计使无人机学会了在观测效率、飞行安全和能耗之间取得平衡。

3. **高度维度的决策具有实际意义**：无人机学会了在接近火区时降低高度以获取详细观测，在远离火区时升高以扩大观测范围，体现了高度-信息量-安全性的三方权衡。

4. **动态环境凸显RL优势**：与静态路径规划不同，火势不断蔓延要求策略具有实时适应性，PPO的策略网络能够根据当前观测动态调整行为。

### 4.2 未来展望

1. **多无人机协同**：扩展为多智能体RL（MAPPO/QMIX），实现多机分工协作，提升大面积火场的监测效率

2. **更精细的火势模型**：引入Rothermel物理模型或基于CNN的火势预测，提高火场模拟的物理真实性

3. **固定翼无人机模型**：加入最小速度和转弯半径约束（Dubins Car模型），更贴近实际航空器

4. **迁移学习与域适应**：在仿真环境训练后迁移到真实火场数据，缩小sim-to-real gap

5. **通信约束下的分布式决策**：考虑无人机间通信延迟和丢包，研究有限通信下的协同策略

6. **人机协同**：引入人类专家的先验知识（如已知重点保护区），实现人机混合决策
